## 목적

SPEC.md의 M0 MVP 스코프에 따라 HQ 운영 관리자를 위한 관리자 대시보드를 구현합니다. 모든 지점의 KPI와 이슈를 모니터링하고, 정책을 배포할 수 있는 인터페이스를 제공합니다.

## 완료 기준

### 1. 전체 지점 KPI 대시보드

- [ ] 대시보드 레이아웃 구성 (MUI 기반)
- [ ] 전체 지점 요약 통계 카드 컴포넌트
  - 총 매출, 평균 근태율, 재고 부족 지점 수
- [ ] 지점별 KPI 목록 테이블 컴포넌트
  - 정렬, 필터링 기능
- [ ] GraphQL Query 구현 (GetAllStoresDashboard)
- [ ] 실시간 데이터 갱신 (폴링 또는 refetch)
- [ ] 로딩 상태 및 에러 처리

### 2. 이상 징후 알림 표시

- [ ] 알림 목록 컴포넌트 구현
  - 무단결근, 재고 부족, 매출 급감 알림
- [ ] 알림 상세 모달/페이지 구현
- [ ] 알림 심각도별 색상 구분
- [ ] 알림 확인 처리 기능
- [ ] GraphQL Query/Mutation 연동

### 3. 지점별 상태 요약

- [ ] 지점 카드 컴포넌트 구현
  - 지점명, 상태, 주요 지표 표시
- [ ] 지점 상세 페이지 구현
  - 근태, 재고, 매출 탭 분리
- [ ] 지점별 필터링 및 검색 기능
- [ ] GraphQL Query 연동 (GetStoreDetails)

### 4. 정책 변경 인터페이스

- [ ] 정책 설정 페이지 구현
  - 근태 승인 규칙, 재고 임계치 조정
- [ ] 정책 변경 폼 컴포넌트
- [ ] 정책 변경 이력 표시
- [ ] GraphQL Mutation 연동 (UpdatePolicy)

### 5. 주간/월간 리포트 조회

- [ ] 리포트 페이지 구현
- [ ] 리포트 기간 선택 컴포넌트
- [ ] 리포트 데이터 시각화 (차트, 그래프)
- [ ] 리포트 다운로드 기능 (PDF/Excel)
- [ ] GraphQL Query 연동 (GetReports)

## 산출물

- React 컴포넌트 코드
- GraphQL 쿼리/뮤테이션 정의
- UI 컴포넌트 스토리북 문서
- 사용자 가이드 문서

## 검증

- [ ] 컴포넌트 단위 테스트 작성
- [ ] 통합 테스트 작성 (GraphQL 연동)
- [ ] E2E 테스트 작성 (Cypress)

- [ ] 코드 리뷰 완료
