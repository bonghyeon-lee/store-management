### 주요 컴포넌트

1. **Overview KPI Cards**
   - 전체 출근률 (전일 대비 변화율)
   - 전체 매출 (전일 대비 변화율)
   - 전체 안전재고 준수율
   - 알람 건수 (최근 24시간)

2. **지점별 종합 점수 테이블**
   - 정렬: 종합점수 내림차순
   - 필터: 기간 선택, 지점 선택
   - 상태 표시: 정상(녹색), 주의(노랑), 경고(빨강)

3. **일별 출근률 추이 차트**
   - 기간: 최근 30일
   - 라인: 전체 평균, 상위/하위 지점

4. **일별 매출 추이 차트**
   - 기간: 최근 30일
   - 라인: 전체 매출, 전년 동기 대비

5. **알람 현황 테이블**
   - 최근 24시간 알람
   - 필터: 심각도, 상태, 지점

## 기술 스택 제안

### 프론트엔드

- **차트 라이브러리**: Recharts, Chart.js, 또는 Apache ECharts
- **테이블**: Material-UI DataGrid 또는 AG-Grid
- **상태 관리**: Redux Toolkit Query (서버 상태), Apollo Client (GraphQL)

### 백엔드

- **GraphQL API**: 기존 Federation Gateway 활용
- **집계 데이터**: 배치 작업으로 사전 계산하여 캐시 (Redis)
- **실시간 업데이트**: GraphQL Subscription 또는 WebSocket

### 데이터 파이프라인

- **ETL**: dbt 또는 Dagster
- **데이터 웨어하우스**: TimescaleDB (시계열), BigQuery/Snowflake (OLAP)
- **시각화**: Metabase/Superset 또는 Looker Studio (향후)

## 우선순위

### Phase 1 (MVP)

1. HQ 운영 관리자 대시보드 - Overview KPI Cards
2. 지점 점장 대시보드 - 오늘의 KPI Cards
3. 근태 승인 대기 목록
4. 재고 부족 알림

1. 차트 및 그래프 시각화
2. 지점별 비교 분석
3. 프로모션 ROI 분석

### Phase 3

1. 고급 분석 대시보드
2. 예측 모델 통합
3. 커스텀 리포트 생성
